# 为每个页面的路由配置，增加 icon 图标

## 1. 任务要求

我需要你帮我准备好来自 iconify 的 icon 标签。

### 1.1 处理范围

- 文件范围： `src/router/main/**/index.ts` 全部名称为 `index.ts` 的路由文件，仅仅处理路由文件。
- icon 图标来源： iconify
- icon 图标格式： `图标集:图标名` ，**必须满足**该格式。
- 可参考的数据集合： `@iconify/json` 数据包。

### 1.2 实施步骤

我需要你遍历需要处理的路由文件，你仅仅只需要阅读每一个 `index.ts` 的 routes 路由配置数组即可。按照以下步骤来实现 meta.icon 新增或替换：

1. 阅读每一个 `index.ts` 的 definePage 路由配置数组。
2. 阅读 meta.title 或 meta.label 标题。
3. 根据每一个路由配置的标题，自己去寻找语义化强的，最满足标题的 icon。
4. 新增或替换掉 meta.icon 图标。
5. 对于已经有设置好 `mdi` 图标集的 meta.icon 图标，请不要做处理，跳过并为没有设置 icon 的路由做设置。

## 2. 设计要求

### 2.1 图标偏好设置

1. 图标集偏好： 我没有明显的偏好。你看情况来即可。你喜欢什么就用什么吧。优先使用 `mdi` 图标集，你也可以选择其他的图标集。
2. 图标风格一致性： 优先弄**实心风格**的图标，本身无颜色的图标。
3. 页面层级处理：
   - 父级页面和子页面的图标选择，没有区别。
4. 处理范围确认：
   - 没有默认图标时，就提供默认图标。由你自己设计一款 默认图标 对应的 iconify 标签取值。
   - 只需要处理 meta.icon 。其他配置**绝对不做处理**。
   - 不要删除删改路由配置文件本来就有的注释。你只负责处理 meta.icon 这一部分。
5. 特殊页面处理： 不考虑什么平衡图标的表现力和简洁性。我不考虑这个方向，我考虑的是至少提供**有差异性的**，**实心风格**的图标。
